# 🐛 如何查看调试日志

## 问题：看不到日志输出

如果你点击了"生成 XAML"但看不到任何调试日志，请按照以下步骤操作：

---

## ✅ 方法 1：查看正确的日志位置

### 步骤 1：停止当前调试会话
按 **Shift+F5** 停止当前运行的应用。

### 步骤 2：打开调试控制台
在 VS Code 底部，确保选中 **DEBUG CONSOLE** 标签页（不是 TERMINAL）。

### 步骤 3：重新启动调试
按 **F5**，选择 **🌐 Blazor Console (Web UI)**

### 步骤 4：测试生成
1. 浏览器打开 `https://localhost:5001/ui-generation`
2. 输入：`创建一个登录表单，包含用户名、密码输入框和登录按钮`
3. 点击 **"⚙ 生成 XAML ⚙"**

### 步骤 5：查看日志
在 **DEBUG CONSOLE** 中应该看到：

```
🔧 UIService 初始化中...
✅ UIService 已就绪
📝 [DEBUG] 清理后的 XAML 长度: xxx 字符
[DEBUG] XAML 开头: <Grid...
[VALIDATION] 根元素: Grid
✅ [VALIDATION] 成功：XAML 语法有效
```

或者如果失败：

```
❌ [VALIDATION] 失败：第 X 行, 第 Y 列: ...
[DEBUG] XAML 内容预览: ...
```

---

## 🔍 方法 2：检查浏览器控制台

如果 DEBUG CONSOLE 也看不到，尝试：

1. 打开浏览器（Edge/Chrome）
2. 按 **F12** 打开开发者工具
3. 查看 **Console** 标签页
4. 点击"生成 XAML"
5. 查看是否有 JavaScript 错误

---

## 📊 方法 3：查看 Blazor 应用日志

在浏览器的 UI Generation 页面：

1. 点击生成后
2. 查看页面上显示的状态消息：
   - "✓ XAML 生成成功！"
   - "⚠ XAML 已生成但验证失败"
   - "生成失败: ..."

3. 如果显示验证失败，页面会显示具体的错误信息

---

## 🎯 快速截图步骤

如果上述方法都不行，请截图以下内容发给我：

1. **VS Code 界面**：
   - 显示打开的 DEBUG CONSOLE 标签
   - 运行状态（是否在调试）

2. **浏览器界面**：
   - UI Generation 页面
   - 点击生成后的状态消息
   - F12 开发者工具的 Console 标签

3. **错误信息**：
   - 页面上显示的任何错误信息
   - "验证失败" 下方的具体错误描述

这样我可以更准确地帮你诊断问题！
